Chapter 2 — Understanding Your Real Sensors
SHT31 — Temperature and Humidity Sensor (PKR 1,850)
The SHT31 measures temperature and humidity at the same time. It is much more
accurate than the DHT22 we used in simulation. It communicates using I2C — the same
two wires (SDA and SCL) that the OLED screen uses. So it shares those wires with no
problem.
It can measure temperature from -40 to 125 degrees C, and humidity from 0% to 100%.
For mushroom growing you need temperature between 20-28 degrees C and humidity
above 75%.

BH1750 — Light Intensity Sensor (PKR 280)
The BH1750 measures light brightness in lux. It is a very small and cheap chip but gives
very accurate readings from 1 lux (almost dark) up to 65,535 lux (bright sunlight). It also
uses I2C — so it shares the same SDA and SCL wires as the SHT31 and OLED.

ADS1115 — 16-bit Analog to Digital Converter (PKR 800)
The ADS1115 is a helper chip. Your pH sensor and soil moisture sensor produce analog
voltage signals. The ADS1115 reads those voltages with very high precision (16-bit = 0 to
32,767 steps) — much better than the ESP32's own analog pins which only give 12-bit (0
to 4,095 steps).
The ADS1115 connects to the ESP32 via I2C, and the pH and soil sensors connect directly
into the ADS1115 input channels (A0 and A1).

SEN0161 — pH Sensor Kit (PKR 11,900)
The SEN0161 has two parts: a glass probe and a signal board. You dip the glass probe
into water or liquid. The signal board converts the pH reading into a voltage between 0 and
3.3V. That voltage goes into ADS1115 channel A0, and the ESP32 reads it.
pH 7 = neutral (pure water). Below 7 = acidic. Above 7 = alkaline. Mushrooms grow best in
pH 5.5 to 7.5.
This sensor MUST be calibrated with your pH buffer powder before it gives accurate
readings. See Chapter 9 for calibration steps.

SEN0193 — Soil Moisture Sensor (PKR 2,380)
The SEN0193 has two metal probes that you push into the mushroom growing substrate
(the material mushrooms grow in). It is capacitive — meaning it does NOT corrode over
time. Wet substrate gives a low voltage. Dry substrate gives a high voltage. The signal
goes into ADS1115 channel A1.

MH-Z19C — CO2 Sensor (PKR 7,500)
This sensor measures carbon dioxide (CO2) in the air. It works using infrared light — CO2
absorbs infrared, and the sensor measures how much was absorbed.
IMPORTANT — this sensor is VERY different from the simulation! In Wokwi we read
a potentiometer with analogRead(). The real sensor communicates using UART
serial protocol — it sends digital text data through TX and RX wires. The code is
completely different for reading it. See Chapter 7 for the updated code.

The sensor needs 5V power (not 3.3V). Healthy outdoor air = 400 ppm CO2. Fan turns on
above 2,000 ppm. Alarm sounds above 5,000 ppm.

Chapter 3 — The I2C Bus: 4 Sensors, 2 Wires
Four of your components all use I2C communication: the SHT31, the BH1750, the
ADS1115, and the OLED display. I2C is clever — all four devices share the SAME two
wires. You do not need separate wires for each device.

How Does I2C Work?
Every I2C device has a unique address — like a house number. The ESP32 shouts a
message on the shared wire (SDA) and includes the address at the start. Only the device
with that address replies. The other devices on the same wire ignore the message.

Chapter 6 — Relay Module Wiring
The 4-channel relay module is the bridge between your low-power ESP32 and the
high-power devices (fan, fogger, LED strip). The relay has two sides: the control side
(connects to ESP32) and the switch side (connects to the actual devices).
IMPORTANT — Relay Logic is Inverted!

Most opto-isolated relay modules (like yours) are ACTIVE LOW. This means: sending
LOW (0) from ESP32 turns the relay ON, and sending HIGH (1) turns it OFF. This is the
opposite of an LED! The code in Chapter 7 already handles this correctly.

Part 4 of 6 — Reading All Sensors
void readAllSensors() { // SHT31: Temperature and Humidity temperature = sht31.readTemperature();
humidity = sht31.readHumidity(); if (isnan(temperature)) temperature = 0; if (isnan(humidity)) humidity = 0;
// BH1750: Light intensity in lux luxValue = lightMeter.readLightLevel(); if (luxValue < 0) luxValue = 0; //
ADS1115 channel A0: pH Sensor (SEN0161) int16_t rawPH = ads.readADC_SingleEnded(0); float voltPH
= ads.computeVolts(rawPH); // Formula: pH 7 = 2.5V, every 0.18V change = 1 pH unit phValue = 7.0 +
((2.5 - voltPH) / 0.18) + pH_offset; phValue = constrain(phValue, 0.0, 14.0); // ADS1115 channel A1: Soil
Moisture (SEN0193) int16_t rawSoil = ads.readADC_SingleEnded(1); float voltSoil =
ads.computeVolts(rawSoil); // SEN0193: ~3.0V = completely dry, ~1.2V = fully wet soilPercent =
map((int)(voltSoil * 100), 120, 300, 100, 0); soilPercent = constrain(soilPercent, 0, 100); // MH-Z19C: CO2
in parts per million (ppm) co2PPM = myMHZ19.getCO2(); if (co2PPM <= 0) co2PPM = 400; // Print
everything to Serial Monitor Serial.println("-----------------------------------"); Serial.printf("Temperature : %.1f
C\n", temperature); Serial.printf("Humidity : %.1f %%\n", humidity); Serial.printf("CO2 : %d ppm\n",
co2PPM); Serial.printf("pH : %.2f\n", phValue); Serial.printf("Soil : %.0f %%\n", soilPercent);
Serial.printf("Light : %.1f lux\n", luxValue); Serial.println("-----------------------------------"); }

Part 5 of 6 — Controlling Actuators and Updating Display
void controlActuators() { // FAN: Turn ON if CO2 too high OR temperature too high if (co2PPM >
CO2_FAN_ON || temperature > TEMP_MAX) { digitalWrite(PIN_FAN, RELAY_ON); Serial.println(">>
FAN is ON"); } else { digitalWrite(PIN_FAN, RELAY_OFF); } // FOGGER: Turn ON if humidity is too low if
(humidity < HUMIDITY_MIN) { digitalWrite(PIN_FOGGER, RELAY_ON); Serial.println(">> FOGGER is
ON"); } else { digitalWrite(PIN_FOGGER, RELAY_OFF); } // GROW LIGHT: Toggle every 12 hours
automatically if (millis() - lastLight >= LIGHT_CYCLE_MS) { lightOn = !lightOn; lastLight = millis();
digitalWrite(PIN_LIGHT, lightOn ? RELAY_ON : RELAY_OFF); Serial.printf(">> LIGHT is %s\n", lightOn ?
"ON" : "OFF"); } // BUZZER ALARM: Critical emergency conditions if (co2PPM > CO2_ALARM ||

Serial.println("!! CRITICAL ALARM !!"); } } void updateOLED() { display.clearDisplay();
display.setTextSize(1); display.setCursor(0, 0); display.println("= Mushroom FYP ="); display.setCursor(0,
10); display.printf("T:%.1fC H:%.0f%%", temperature, humidity); display.setCursor(0, 20);
display.printf("CO2: %d ppm", co2PPM); display.setCursor(0, 30); display.printf("pH:%.1f Soil:%.0f%%",
phValue, soilPercent); display.setCursor(0, 40); display.printf("Light: %.0f lux", luxValue);
display.setCursor(0, 52); display.printf("F:%s Fg:%s L:%s", digitalRead(PIN_FAN) == RELAY_ON ? "ON "
: "OFF", digitalRead(PIN_FOGGER) == RELAY_ON ? "ON " : "OFF", digitalRead(PIN_LIGHT) ==
RELAY_ON ? "ON " : "OFF"); display.display(); }

publishMQTT() { if (!mqtt.connected()) reconnectMQTT(); StaticJsonDocument<300> doc;
doc["temperature"] = temperature; doc["humidity"] = humidity; doc["co2"] = co2PPM; doc["ph"] = phValue;
doc["soil"] = soilPercent; doc["lux"] = luxValue; doc["fan"] = (digitalRead(PIN_FAN) == RELAY_ON);
doc["fogger"] = (digitalRead(PIN_FOGGER) == RELAY_ON); doc["light"] = (digitalRead(PIN_LIGHT) ==
RELAY_ON); char payload[300]; serializeJson(doc, payload); mqtt.publish(mqtt_topic, payload);
Serial.println(">> MQTT data published"); } // --- Main Loop --- void loop() { mqtt.loop(); unsigned long now
= millis(); // Read sensors and update display every 3 seconds if (now - lastRead >= 3000) { lastRead =
now; readAllSensors(); controlActuators(); updateOLED(); } // Publish to MQTT every 10 seconds if (now lastMQTT >= 10000) { lastMQTT = now; publishMQTT(); } }

Chapter 8 — How to Flash Code to Your ESP32
Step 1 — Install Arduino IDE
Download Arduino IDE from arduino.cc/en/software and install it on your computer.
Step 2 — Add ESP32 Board Support
Open Arduino IDE. Go to File → Preferences. In the 'Additional Board Manager URLs' box,
paste this link and click OK: https://raw.githubusercontent.com/espressif/arduino-esp32/gh
-pages/package_esp32_index.json Then go to Tools → Board → Board Manager. Search
for esp32 (by Espressif Systems). Click Install. Wait a few minutes.
Step 3 — Select the Right Board
Go to Tools → Board → ESP32 Arduino → select 'ESP32 Dev Module'. Then go to Tools
→ Port → select the COM port that appears when you plug in your ESP32.
Step 4 — Install All Libraries
Go to Tools → Manage Libraries. Search for and install each library listed in Chapter 7.
Step 5 — Paste and Edit the Code
Create a new sketch (File → New). Delete the default code. Paste the complete code from
Chapter 7. Find the two WiFi lines near the top and change YOUR_WIFI_NAME and
YOUR_WIFI_PASS.
Step 6 — Upload
Click the Upload button (arrow pointing right). If it shows 'Connecting...' and stops — hold
the BOOT button on your ESP32 while clicking Upload, then release BOOT when

Chapter 9 — pH Sensor Calibration
The SEN0161 pH sensor must be calibrated before use. Without calibration, readings
could be wrong by 1 to 2 pH units. You have pH 4.01 and pH 6.86 buffer powder in your
BOM — here is exactly how to use them.
1. Dissolve the pH 6.86 buffer powder in 250 ml of clean water. Stir well.
2. Rinse the glass pH probe with clean distilled water.
3. Dip the glass probe into the pH 6.86 buffer solution.
4. Wait 2 to 3 minutes for the reading to stabilize.
5. Open the Serial Monitor and look at the pH value being printed.
6. If the reading shows exactly 6.86 — great! No adjustment needed.
7. If the reading is different — find the small blue trimmer pot on the SEN0161 board.
8. Use a small flat screwdriver to slowly turn the trimmer pot clockwise or anti-clockwise.
9. Watch the Serial Monitor. Stop turning when it reads 6.86.
10. Rinse the probe. Now test with pH 4.01 buffer solution to verify accuracy.
11. If the pH 4.01 reading is off by a fixed amount, use the software offset below.

Software Fine-Tuning (if still slightly off)
If after hardware adjustment the readings are still a little off, you can fix it in code by
changing this one line:
float pH_offset = 0.0; // Change this number

Example: if pH 6.86 buffer reads as 7.10 in Serial Monitor, then change pH_offset to -0.24
(because 6.86 minus 7.10 equals -0.24). Upload the code again and the readings will shift
down by 0.24.
